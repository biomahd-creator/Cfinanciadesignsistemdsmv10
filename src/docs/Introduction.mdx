# CESIONBNK Design System

Bienvenido al **Design System de CESIONBNK** — la plataforma financiera con modulo de Factoring.

## Arquitectura de Componentes

El DSM se organiza en **4 capas** siguiendo principios de Atomic Design:

| Capa | Directorio | Descripcion | Ejemplos |
|------|-----------|-------------|----------|
| **1. UI** | `/components/ui/` | Atomos y moleculas. Componentes base sobre Radix UI + Tailwind | Button, Badge, Input, Card, Dialog |
| **2. Advanced** | `/components/advanced/` | Organismos complejos reutilizables | DataTable, Combobox, Timeline, GaugeChart |
| **3. Patterns** | `/components/patterns/` | Organismos de negocio. Componen vistas completas | FactoringKpiCard, ApprovalFlowWizard, NotificationCenter |
| **4. Widgets** | `/components/widgets/` | Bloques funcionales especificos del dominio | StatCard, StatusKPICard, FilterBar |

## Modulos Adicionales

| Modulo | Directorio | Contenido |
|--------|-----------|-----------|
| **Factoring** | `/factoring/` | Componentes de dominio: InvoiceCard, PayorCard, RiskIndicator, etc. |
| **Accessibility** | `/components/accessibility/` | SkipLink, FocusTrap, LiveRegion |
| **Help** | `/components/help/` | HelpCenter, ProductTour, ContextualHelp |

## Stack Tecnologico

- **React 18** + TypeScript
- **Tailwind CSS v4** con tokens semanticos
- **Radix UI** + **Shadcn/UI** para componentes base
- **Recharts** para visualizacion de datos
- **Motion** para animaciones
- **Storybook 7.x** para documentacion

## Convenciones Clave

1. **`cn()` helper** vive en `/components/ui/utils.ts` (no en `/lib/utils`)
2. **Tokens semanticos** deben definirse en 3 lugares: `:root`, `.dark`, y `@theme`
3. **Badges en tablas de factoring** usan estilo **Soft Outline** del DSM
4. **TabsList** sigue patron constrained: `grid w-full grid-cols-N max-w-lg`
5. **Variables `--cfinancia-*`** son canales RGB y usan `rgb(var(...))`
