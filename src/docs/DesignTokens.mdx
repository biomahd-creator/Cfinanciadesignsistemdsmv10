# Design Tokens

Los tokens semanticos del DSM se definen en `/styles/globals.css` y se exponen via Tailwind CSS v4.

## Colores de Marca

| Token | Hex | Uso |
|-------|-----|-----|
| `--primary` | `#00c951` | Verde principal. CTAs, acciones primarias, focus ring |
| `--primary-foreground` | `#ffffff` | Texto sobre primary |
| `--secondary` | `#1C2D3A` | Navy principal. Headings, acciones secundarias |
| `--secondary-foreground` | `#ffffff` | Texto sobre secondary |

### C-Financia Brand

| Token | Hex | Uso |
|-------|-----|-----|
| `--cfinancia-accent` | `#DEFB49` | Amarillo accent C-Financia |
| `--cfinancia-accent-hover` | `#43F977` | Hover verde accent |
| `--cfinancia-navy` | `#052937` | Navy oscuro C-Financia |

> Las variables `--cfinancia-*` son canales RGB y se usan como `rgb(var(--cfinancia-accent))`.

## Colores Semanticos

### Light Mode

| Token | Tailwind | Hex |
|-------|----------|-----|
| `--background` | `bg-background` | `#ffffff` |
| `--foreground` | `text-foreground` | `#1C2D3A` |
| `--card` | `bg-card` | `#ffffff` |
| `--muted` | `bg-muted` | `#f4f4f5` |
| `--muted-foreground` | `text-muted-foreground` | `#52525b` |
| `--accent` | `bg-accent` | `#f4f4f5` |
| `--destructive` | `text-destructive` | `#dc2626` |

### Dark Mode

Los mismos tokens con valores ajustados. La clase `.dark` en `<html>` activa el modo oscuro.

## Tipografia

| Propiedad | Valor |
|-----------|-------|
| Font family | `Satoshi`, sans-serif |
| Letter spacing | `0.025em` (tracking-wide) |

> No usar clases de Tailwind para font-size (`text-2xl`), font-weight (`font-bold`) o line-height (`leading-none`) a menos que el usuario lo solicite.

## Spacing

Se usa la escala de Tailwind (4px base):
- `gap-1` = 4px
- `gap-2` = 8px
- `gap-4` = 16px
- `gap-6` = 24px
- `gap-8` = 32px

## Border Radius

| Token | Tailwind | Valor |
|-------|----------|-------|
| `--radius` | `rounded-lg` | `0.5rem` |
| Variaciones | `rounded-md`, `rounded-xl`, `rounded-2xl` | 0.375rem, 0.75rem, 1rem |

## Regla de 3 Lugares

Cuando se agrega un nuevo token semantico, debe ir en **3 lugares**:
1. `:root { --new-token: value; }`
2. `.dark { --new-token: dark-value; }` con puente `--color-*`
3. `@theme { --color-new-token: var(--new-token); }`

## Excepciones de Color

Estos archivos usan colores hardcodeados intencionalmente — **no tocar**:
- `code-block.tsx` — Sintaxis highlighting
- `ColorPresetButton.tsx` — Previews de color
- `bg-white/10|20` sobre fondos oscuros
- Hex en `FactoringKpiCard.tsx` — Variantes por hex
- Archivos en `/imports/` — Componentes importados de Figma
